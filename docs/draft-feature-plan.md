# 📝 입찰공고 제안서 초안 작성 기능 기획서

## 1. 개요

입찰공고 리스트에서 **원클릭으로 AI가 제안서 초안을 생성**해주는 기능입니다.
공고의 제목·기관·예산 등 기본 정보만으로 Gemini AI가 섹션별 초안을 작성합니다.

> ⚠️ AI가 생성하는 초안은 **참고용**이며, 실제 제출 전 반드시 전문가 검토가 필요합니다.

---

## 2. 사용자 흐름 (User Flow)

```
공고 리스트 → [✍️ 초안 작성] 클릭 → AI 초안 모달 열림 → 로딩(10~30초)
→ 초안 결과 표시 → [📋 복사] 또는 [🔄 다시 생성]
```

### 단계별 상세

| 단계 | 화면 | 설명 |
| :--- | :--- | :--- |
| 1 | 공고 카드 | 각 공고 하단의 `✍️ 초안 작성` 버튼 클릭 |
| 2 | 상세 모달 | 또는 모달 내 `✍️ 초안 작성하기` 버튼 클릭 |
| 3 | 초안 모달 | 로딩 애니메이션 + "AI가 초안을 작성하고 있습니다" 안내 |
| 4 | 결과 표시 | 마크다운 형식의 제안서 초안이 렌더링됨 |
| 5 | 후속 액션 | 복사 / 다시 생성 / 닫기 |

---

## 3. AI 초안 생성 구조

### 입력 (공고 정보)
- 공고명, 공고기관, 수요기관
- 입찰방법, 계약방법
- 추정가격, 입찰마감일

### 출력 (제안서 초안의 6개 섹션)

| 섹션 | 내용 | 분량 |
| :--- | :--- | :--- |
| **1. 사업 이해** | 사업 배경·목적 분석, 발주기관 니즈 파악 | 200~300자 |
| **2. 추진 전략** | 핵심 전략, 차별화 포인트 | 300~400자 |
| **3. 수행 방안** | 단계별 수행 계획, 방법론 | 400~500자 |
| **4. 추진 일정** | 단계별 일정표 (표 형식) | 표 |
| **5. 기대 효과** | 정량적/정성적 효과 | 200~300자 |
| **6. 수행 조직** | 투입 인력 구성안 | 200자 |

### 사용 모델
- **Gemini 2.5 Pro** (`gemini-2.5-pro-preview-05-06`)
- 최고 성능 모델로 전문적이고 구체적인 초안 생성

---

## 4. 기술 구현

```
[프론트엔드] page.tsx
  └─ ✍️ 초안 작성 버튼 클릭
  └─ POST /api/generate-draft (공고 정보 전송)
  └─ 초안 모달에서 결과 렌더링

[백엔드] /api/generate-draft/route.ts
  └─ 요청 검증 → Gemini API 호출 → 마크다운 응답 반환

[AI] Gemini 2.5 Pro
  └─ 공고 정보 + 프롬프트 → 섹션별 제안서 초안 생성
```

---

## 5. 향후 고도화 계획

### Phase 2: 회사 프로필 연동
- 사전 등록된 회사 정보(실적, 역량, 인력)를 프롬프트에 반영
- 더 개인화된 맞춤형 초안 생성

### Phase 3: 첨부 파일 분석
- 공고 첨부 제안요청서(RFP) PDF를 업로드하여 분석
- RFP의 평가 기준에 맞춰 초안 최적화

### Phase 4: HWPX 문서 출력
- 생성된 초안을 HWPX(한글) 파일로 변환
- 제안서 템플릿에 자동 삽입하여 다운로드

---

## 6. 주의사항

> [!CAUTION]
> - AI 초안은 공고 제목만으로 추론하는 **참고 자료**입니다
> - 실제 RFP(제안요청서)를 반영하지 않으므로 세부 요구사항은 별도 확인 필요
> - **반드시 전문가 검토 후** 실제 제안서에 활용하세요
